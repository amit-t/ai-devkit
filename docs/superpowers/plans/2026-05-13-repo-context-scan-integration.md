## Goal

Make `init.wb` and `join.wb` automatically build wb-owned domain context for every cloned source repo, and surface a `wb.rescan` command for manual refresh / failure recovery. After this plan lands:

- Every new workbench gets a `${WB_DIR}/context/<name>/CONTEXT.md` (+ seeded ADRs) per registered source repo, plus an aggregate `${WB_DIR}/context/README.md` index.
- Source repos under `repos/<name>/` are never mutated — scans run in a throwaway `git worktree` sandbox.
- Scan failures degrade gracefully — a stub file is written with `status: scan-failed` + retry instructions. Init never aborts because of a scan.
- `wb.rescan <repo>` / `wb.rescan --all` repairs failures and refreshes context after source-repo drift.
- `devkit doctor` extends to verify skill vendoring, symlinks, library presence, engine env, and (when run inside a stamped wb) context-dir health.

## Architecture

Three layers:

| Layer | Responsibility | Location |
|-------|---------------|----------|
| **Vendored skill** | LLM-driven semantic scan of a single repo. Generic, unchanged from upstream `at-skills`. | `${DEVKIT_DIR}/skills/repo-context-scan/` (rsync mirror) |
| **Devkit wrapper (shell, deterministic)** | Worktree sandbox setup, force-fresh pre-wipe, harvest, frontmatter, stub-on-failure, aggregate index. | `${DEVKIT_DIR}/lib/wb-context-scan.zsh` |
| **Orchestration (agent prompts + CLIs)** | Loops in `init.prompt.md` + `join.prompt.md`; standalone `wb.rescan` CLI; one-line hint from `register-repo.sh`. | Multiple touch points (manifest below) |

### Output layout per wb

```
${WB_DIR}/
  context/
    README.md                          # aggregate index, generated by wrapper
    payments-svc/
      CONTEXT.md                       # devkit frontmatter + skill-produced body
      docs/adr/0001-*.md
    billing-svc/
      CONTEXT.md                       # may be a stub if scan failed
    e2e-tests/
      CONTEXT-MAP.md                   # multi-context repo variant
      api/CONTEXT.md
      ui/CONTEXT.md
  .context-scan/                       # worktree staging — gitignored, transient
  .workbench-manifest.json             # gains "auto_generated" key (template patch)
```

### Per-repo scan flow

```
setup (shell, deterministic)
  ├─ git worktree prune + rm -rf .context-scan/<name>       # defensive
  ├─ mkdir -p .context-scan/<name>
  ├─ git -C repos/<name> worktree add --detach .context-scan/<name> HEAD
  ├─ rm worktree's CONTEXT.md / CONTEXT-MAP.md / docs/adr   # force-fresh
  ├─ flock acquire .context-scan/.lock
  └─ print SCAN_DIR=.context-scan/<name>     # consumed by caller

scan (LLM, agent dispatch — three flavors)
  ├─ Claude in-session: parent agent's Task tool → sub-agent at SCAN_DIR, /repo-context-scan
  ├─ Devin in-session:  inline /repo-context-scan invocation in init/join agent
  └─ Standalone (wb.rescan): claude -p / devin -p subprocess fires SCAN_DIR

finalize (shell, deterministic)
  ├─ if CONTEXT.md or CONTEXT-MAP.md produced → harvest to context/<name>/ + prepend frontmatter
  ├─ else (or --fail-reason given) → write stub CONTEXT.md with failure frontmatter + body
  ├─ git worktree remove --force .context-scan/<name>
  └─ flock release

aggregate (shell, deterministic, one pass per scan batch)
  ├─ read project.conf REPOS array
  ├─ walk context/*/CONTEXT.md (and CONTEXT-MAP.md)
  ├─ regenerate context/README.md table: Repo | Role | Status | Top concepts
  └─ status values: scanned | scan-failed | orphan | missing
```

### Engine selection

- New env var `DEVKIT_DEFAULT_ENGINE=devin|claude` written to `~/.zprofile` by `install.zsh` next to `DEVKIT_CLONE`.
- Default: `devin` if `command -v devin` succeeds, else `claude`.
- All callers honor it. `--agent devin|claude` flag overrides per-invocation. Mirrors existing `init.zsh / join.zsh / update.zsh` flag convention.

### Failure policy

- Per-repo scan failure: skip + write stub. Init / join always continue.
- Stub frontmatter records `status: scan-failed`, `fail_reason`, `failed_at`, `retry_with: "wb.rescan <name>"`.
- `wb.rescan <name>` repairs (auto-wipes stubs); `wb.rescan --all` re-attempts everything.
- `--force` blows away user-authored prose; default preserves it via skill's idempotent re-run mode.
- `--aggregate-only` regenerates the index without rescanning.
- wb.rescan self-commits per run; **no auto-push** — user reviews + pushes.

### CONTEXT.md frontmatter (locked Q15+16)

```yaml
---
generated_by:   ai-devkit/repo-context-scan
generated_at:   2026-05-13T14:22:18Z
source_repo:    payments-svc
source_url:     https://github.com/foo-org/payments-svc
source_commit:  a1b2c3d4e5f6...
devkit_version: 1.1.0
skill_version:  7f8a1c2

# failure-only:
fail_reason:    "no HEAD in source repo"
failed_at:      2026-05-13T14:22:18Z
retry_with:     "wb.rescan payments-svc"
---
```

Body on success: skill-produced content untouched.
Body on failure (stub):

```markdown
# <repo-name> (scan failed)

Repo context scan did not produce output. Reason recorded in frontmatter.

Re-run after fixing the underlying issue:

    wb.rescan <repo-name>

If the failure is structural (e.g., the source repo isn't appropriate for
scanning), remove this directory:

    rm -rf context/<repo-name>
```
